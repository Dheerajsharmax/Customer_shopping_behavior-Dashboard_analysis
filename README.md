# Customer_shopping_behavior-Dashboard_analysis
This project analyzes customer shopping behavior using transactional data from 3,900 purchases across various product categories. The goal is to uncover insights into spending patterns, customer segments, product preferences, and subscription behavior to guide strategic business decisions. 


Customer Shopping Behavior Analysis
Overview

This data analytics project focuses on analyzing customer shopping behavior using transactional retail data. The objective is to identify purchasing patterns, customer preferences, and key business insights through data exploration, SQL analysis, and interactive dashboards. The project follows an end-to-end analytics workflow, from data loading and cleaning to visualization and reporting.

Dataset

The dataset contains customer demographic information, purchase details, and shopping behavior attributes.

Key features include:

Customer demographics (Age, Gender, Location, Subscription Status)

Purchase details (Item Purchased, Category, Purchase Amount, Season, Size, Color)

Shopping behavior (Discount Applied, Payment Method, Shipping Type, Review Rating, Frequency of Purchases)

The dataset was reviewed for missing values, errors, and inconsistencies before analysis.

Tools & Technologies

SQL (PostgreSQL / MySQL / SQL Server) – Querying and data analysis

Power BI – Interactive dashboard creation

Gamma – Presentation (PPT) creation

GitHub – Version control and project documentation

Steps Performed

Loaded the dataset using Python

Performed Exploratory Data Analysis (EDA)

Handled missing values and removed data errors

Created required calculated and derived columns

Executed SQL queries to answer business questions

Built an interactive Power BI dashboard

Created a detailed analytical report

Designed a presentation summarizing insights using Gamma

Dashboard

The Power BI dashboard provides:

Category and product-level sales insights

Customer segmentation analysis

Purchase trends and behavior patterns

Impact of subscriptions and discounts on sales

The dashboard is designed for easy interpretation by business stakeholders.

Results & Insights

Identified top-performing product categories and items

Analyzed customer purchasing frequency and spending behavior

Evaluated the influence of discounts and subscription status

Highlighted patterns that support data-driven business decisions

How to Run the Project

Clone the repository

Open the Python notebooks/scripts to load and clean the dataset

Run SQL queries using PostgreSQL/MySQL/SQL Server

Open the Power BI file to view the dashboard

Review the report and presentation for summarized insights

Conclusion

This project demonstrates a complete data analytics workflow, combining Python, SQL, and Power BI to transform raw data into meaningful business insights. It showcases practical skills relevant to data analyst and business intelligence roles.
